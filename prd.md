# 🧠 Product Requirements Document (PRD)
## Portfolio Developer — FilipaoVfx

---

## 1. 🎯 Objetivo del Producto

Construir un portfolio interactivo que:

- Maximice la conversión de reclutadores (interés → contacto)
- Demuestre pensamiento de producto + arquitectura técnica
- Muestre 3 proyectos clave como sistemas reales (no demos)
- Refleje una identidad visual fuerte (Glassmorphism + Neobrutalism)
- Comunique proceso, no solo resultados

---

## 2. 👤 Usuario Objetivo

### Primario:
- Reclutadores técnicos (5–30 seg de atención inicial)

### Secundario:
- Tech leads / founders
- Otros desarrolladores

---

## 3. 💡 Propuesta de Valor

"No soy solo un dev que programa, soy alguien que diseña sistemas completos y piensa en producto, performance y escalabilidad."

---

## 4. 🧩 Proyectos a Mostrar

### 1. CrackingWall
- Plataforma de wallpapers (SEO + distribución)
- Stack: Astro + Cloudflare + Supabase
- Enfoque: monetización + performance

### 2. Clippy
- Sistema de clipboard cloud (multi-device)
- Enfoque: realtime + UX invisible
- Insight: comportamiento digital del usuario

### 3. Indexer
- Buscador de comunidad / repos
- Enfoque: clasificación + discovery
- Inspiración: crypto bubbles + graph thinking

---

## 5. 🧠 Concepto UX Central

### “Ruta del desarrollador”

El portfolio no es scroll → es navegación:

- Timeline / workflow (tipo n8n)
- Cada proyecto = nodo
- Cada nodo conecta con:
  - Problema
  - Solución
  - Stack
  - Aprendizaje

---

## 6. 🎨 Dirección de Diseño

### Mix:
- 70% Glassmorphism → profundidad
- 30% Neobrutalism → estructura

### Reglas:
- Contenedores → brutalistas
- Contenido → glass
- Hover → glow + interacción

---

## 7. 🧭 Estructura del Portfolio

### 1. Hero
- Identidad fuerte
- Mensaje claro:
  - “I build systems, not just apps”

### 2. Navegación principal (timeline / board)
- Visual interactivo
- Nodos conectados

### 3. Sección Proyectos (core)
Cada proyecto incluye:

- Contexto
- Problema
- Arquitectura
- Decisiones técnicas
- Demo / preview
- Aprendizajes

### 4. Sección “How I Think”
- Explica tu forma de resolver problemas
- Sistemas, no features

### 5. Call to Action
- Contacto directo
- GitHub + LinkedIn

---

## 8. ⚡ Diferenciadores Clave

- UI inmersiva (no típica)
- Narrativa tipo sistema
- Enfoque en decisiones técnicas
- Pensamiento de producto
- Performance-first mindset

---

## 9. 📊 Métricas de Éxito

- Tiempo en página (>1 min)
- Click en proyectos
- Scroll depth
- Contact clicks

---

## 10. 🚀 Roadmap

### Fase 1:
- MVP funcional (1 semana)

### Fase 2:
- Interacciones avanzadas (timeline + animaciones)

### Fase 3:
- Optimización performance + SEO

---

## 11. 🔥 Mensaje Final

El portfolio debe responder:

> “¿Por qué este dev es diferente?”

Y la respuesta debe ser:

> “Porque piensa como un constructor de sistemas.”
